## 网站地图

- 访问地址：`https://你的域名/sitemap.xml`
- `robots.txt` 已指向：`Sitemap: /sitemap.xml`
- 包含：首页、博客、下载中心、dows.html、全部文章、全部标签页
- 构建时指定域名：`set SITE_DOMAIN=你的域名.com` 后执行 `npm run build`
- 建议在 Bing / Google 站长工具提交该地址

### IndexNow 推送 403（UserForbiddedToAccessSite）

**验证文件已是 HTTP 200 且内容与密钥一致，仍 403** → 说明 txt 部署正确，但 **Bing 站长工具里站点尚未完成所有权验证**，或 `SITE_DOMAIN` 与站长工具里添加的站点不一致（常见：`qhqrl.com` vs `www.qhqrl.com`）。

按下面检查：

1. 打开 [Bing Webmaster Tools](https://www.bing.com/webmasters) → **添加站点**
2. 添加的 URL 必须与 `SITE_DOMAIN` **完全一致**（裸域或 www 二选一，与浏览器访问首页的域名相同）
3. 用 **HTML/XML 文件** 或 **IndexNow 密钥** 验证，部署 `https://域名/{密钥}.txt`（仅一行密钥）
4. 在站长工具点击 **验证**，状态必须为 **已验证**（仅线上能打开 txt 不够，必须在后台点验证通过）
5. GitHub Secret：`SITE_DOMAIN`（无 `https://`）、`INDEXNOW_KEY`（与 txt 文件名、内容相同）
6. 验证通过后等待 **10–30 分钟** 再重跑 Actions
7. 若站长工具添加的是 `www.xxx.com`，而 Secret 是 `xxx.com`：把 `SITE_DOMAIN` 改为 `www.xxx.com`，或新增 Secret `INDEXNOW_HOST=www.xxx.com`
8. 脚本会自动尝试 www / 裸域两种 host；若某一变体成功，日志会提示应固定使用的域名

### 验证文件返回 18828 字节（HTML）而不是 32 字节

说明 `https://域名/密钥.txt` 实际打开的是**首页 HTML**，验证 txt **没有部署到根目录**。

**必须做：Cloudflare Pages 环境变量**

1. Cloudflare → 你的 Pages 项目 → **Settings → Environment variables**
2. 添加（Production）：
   - `INDEXNOW_KEY` = 与 Bing 相同的 32 位密钥
   - `SITE_DOMAIN` = 你的域名（如 `qhqrl.com`）
3. **重新部署**（Retry deployment）
4. 浏览器打开 `https://域名/密钥.txt`，应只看到一行密钥，约 32 个字符
5. 再跑 GitHub Actions

构建命令 `npm run build` 会自动生成 `src/密钥.txt` 并输出到 `_site/密钥.txt`。

## Gemini 自动发文 + 必应 IndexNow 推送

GitHub Secrets 需配置：

| Secret | 说明 |
|--------|------|
| `GEMINI_API_KEY` | Google Gemini API 密钥 |
| `INDEXNOW_KEY` | 必应 IndexNow 密钥（与 txt 文件名一致） |
| `SITE_DOMAIN` | 站点域名，如 `qhqrl.com`（不要带 https） |

流程（`.github/workflows/gemini-auto-content.yml`）：

1. 每天香港时间 **08:12** 自动生成 1 篇文章（cron `12 0 * * *` UTC）
2. 手动：**Actions → Gemini Auto Content Updater → Run workflow** → 输入 1–9 篇
3. 构建 Eleventy，将 `src/{INDEXNOW_KEY}.txt` 复制到 `_site`
4. 提交 `src/posts/`、`keywords.json`、验证 txt 到 GitHub
5. 等待线上验证文件 HTTP 200 后，**仅推送当天新发文 URL**（`PUSH_NEW_ONLY=1`，已默认开启）

### IndexNow 两种模式

| 模式 | 环境变量 | 行为 |
|------|----------|------|
| 仅新文（推荐/CI 默认） | `PUSH_NEW_ONLY=1` | 只推当天 `date:` 的新文章 |
| 全站 | 不设置 | 推送 `_site` 全部页面（约 60+ URL） |

识别当天新文的顺序：

1. `scripts/.indexnow-pending.json`（Gemini 生成后自动写入）
2. `src/posts/*.md` 里 `date:` 等于当天（香港时区）
3. Git 最近一次提交新增的 `src/posts/*.md`

本地手动测试：

```bash
set GEMINI_API_KEY=你的密钥
npm run generate

set SITE_DOMAIN=你的域名.com
set INDEXNOW_KEY=你的密钥
set PUSH_NEW_ONLY=1
npm run build
npm run bing:push
```

### 安装教程视频（懒加载）

仅对 Windows / macOS 等强安装教程，在 front matter 增加：

```yaml
videoEmbed: "https://www.youtube.com/embed/你的视频ID"
videoTitle: 与文章 H1 一致的标题
```

页面会显示「点击加载演示」按钮，点击后才插入 iframe，并输出 VideoObject 结构化数据。未配置 `videoEmbed` 时不显示视频区块。
